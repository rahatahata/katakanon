# 04 — API・リアルタイム契約

本書は [01_requirements.md](./01_requirements.md) を実装するための **HTTP API** および **リアルタイム（例: WebSocket）メッセージ**のたたき台である。パス・フィールド名は実装時に確定する。

---

## 1. 原則

- **認証** — MVP はアカウントなし。ルーム参加時に **表示名** を送り、サーバが `participant_id` を発行する想定。
- **権限** — ホストのみ **ゲーム開始**、ホスト移譲は要件 [§7.8](./01_requirements.md#78-ルーム参加ホスト) に従う。

---

## 2. HTTP API（案）

| メソッド | パス（例） | 説明 |
|----------|------------|------|
| `POST` | `/rooms` | ルーム作成。ボディ: ラウンド数等。レス: `room_code`, `join_url` |
| `GET` | `/rooms/{code}` | ルームメタの参照（存在確認、設定表示用） |
| `POST` | `/rooms/{code}/join` | 参加。ボディ: `display_name`（必須）。レス: `participant_id`, `participant_token`（任意） |
| `POST` | `/rooms/{code}/avatar` | アイコン画像のアップロード（入室後）。要件 [§7.8.3](./01_requirements.md#783-手描きアイコンの扱い) |
| `GET` | `/words` | （管理用）お題マスタ取得。MVP はクライアント同梱でも可 |

静的ファイル配信・CORS・レート制限はデプロイ環境に依存。

---

## 3. WebSocket（または同等）イベント（案）

### 3.1 クライアント → サーバ（コマンド）

| イベント名（例） | ペイロード概要 |
|-------------------|----------------|
| `game:start` | ホストのみ。ゲーム開始 |
| `answer:enqueue` | 解答ボタン |
| `answer:judge` | 説明者: `correct` / `incorrect`、対象 `participant_id` |
| `accuse:start` | 指摘ボタン（先着ロック） |
| `accuse:respond` | 説明者: `admit` / `deny` |
| `vote:cast` | 否認時の可決／否決 |
| `profile:update` | アイコン更新（任意） |

### 3.2 サーバ → クライアント（状態配信）

| イベント名（例） | 内容 |
|-------------------|------|
| `state:full` | ルーム全体スナップショット（参加リスト、スコア、フェーズ） |
| `state:delta` | 差分更新（任意最適化） |

スナップショットに含める情報の例:

- 参加者一覧（名前・アイコン URL・スコア）
- 現在フェーズ（ロビー / ターン中 / 指摘中 / 投票中）
- 説明者 ID、**お題**（説明者以外にはマスク）
- 解答キュー順
- 指摘・投票の途中状態

---

## 4. エラー・バリデーション

- プレイヤーネーム — [§7.8.2](./01_requirements.md#782-プレイヤーネームの入力規約) を **サーバで再検証**。
- 指摘の排他、解答キュー、是認／否認の整合 — サーバが **単一の状態機械**で拒否する。
