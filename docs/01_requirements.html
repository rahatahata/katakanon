カタカナ禁止ゲーム

通話ツール（Discord 等）と併用して遊ぶ Web アプリ型マルチプレイゲーム。カタカナのお題をカタカナなしで説明し、仲間に当てさせる言葉遊び。

セッション型マルチプレイ リアルタイム同期 PC・スマホ対応 Next.js / Fastify / Socket.IO

1 文書の目的

本書は、通話ツール（Discord 等）と併用して遊ぶ カタカナ禁止説明ゲーム を Web アプリとして実装するにあたり、プロダクトの範囲・挙動・用語を共有するための仕様書である。

2 概要

2.1 一言説明

💡
参加者の一人が カタカナのお題 を、カタカナを使わず通話で説明し、他の参加者が正解を当てる。違反疑惑は 指摘と投票 で処理し、得点 で順位をつける セッション型マルチプレイ ゲーム。

2.2 アプリの担当範囲

担当内容
本アプリ ルーム管理、参加者同期、お題の秘匿表示、参加者プロフィール（プレイヤーネーム・手描きアイコン）、順番・ターン・ラウンド・得点、解答・正誤・指摘・投票の手続きと記録
外部（通話ツール） 音声による説明・発話・話し合い
参加者 「カタカナを言ったか」等の ルールの解釈（システムはその内容を定義しない）

2.3 目的・対象

主な利用者
開発者の友人同士での遊び
開発者向け目的
Web 技術の習得・発展、ポートフォリオとしての提示

2.4 プラットフォーム

プラットフォーム
PC・スマートフォン対応（レスポンシブ Web）

3 用語

用語意味
ルーム1 つのゲームセッションの単位。1 ルーム = 1 ゲーム。
参加者ルームに参加しているプレイヤー。
プレイヤーネーム入室時に参加者自身が入力する 表示名。必須。
アイコン（手描き）入室後 に描画 UI で編集・確定するアイコン。未設定でも入室可能（プレースホルダー表示）。
ホストルーム作成者。ゲーム開始権限を持つ。切断時は権限が自動移譲。任意の参加者へ手動移譲も可。
説明者当該ターンにお題を知り、カタカナを使わず説明する役。
解答者説明者以外。正解を推測し解答する。
指摘者説明者のカタカナ使用を疑い、指摘操作を行った参加者。
ターン1 人の説明者が 1 お題 について説明し、解答・指摘・正誤判定などが行われる単位。
ラウンド参加者 全員 が それぞれ 1 回ずつ 説明者を経験するまでのまとまり。1 ラウンド = N ターン（N = 参加者数）。
お題当該ターンの説明者の画面にのみ表示される カタカナ表記のワード（MVP は固定リスト）。
解答キュー解答ボタンが押された順に並ぶ 解答者の待ち行列。先頭が「解答権」を持つ。

4 ルールと判定

4.1 カタカナ禁止の性質

⚠️
違反の内容・境界（何を「カタカナとみなすか」等）は 参加者同士の話し合い に委ねる。本システムはその内容を定義しない。

4.2 システム上の違反処理

🔧
違反の 手続き的な認定 は、指摘ボタン → 説明者の是認・否認 → （否認時のみ）投票 の結果として記録する。

5 参加者人数とモード

5.1 想定（MVP）

ゲーム開始条件
3 人以上のオンライン参加者が必要。3 人未満では開始操作を受け付けない。
2 人プレイ
将来機能 投票なし・別ルール化等を検討。

※ 説明者・指摘者を除いた解答者が投票に参加できない場合、ゲームが成立しないため 3 人以上を条件とする。

7.3 お題の表示

🔒
秘匿表示
お題は 当該ターンの説明者の画面にのみ 表示。他参加者には一切送信しない。
📋
固定リスト（MVP）
サーバ側で固定リストを保持。同一ゲーム内で同じお題は重複出題しない。
🔢
必要お題数
参加者数 × ラウンド数を満たす必要あり。不足する場合はゲーム開始不可。

🔒
指摘フロー中（指摘発生から是認／否認、または投票確定まで）は、解答ボタンおよび指摘ボタンを無効化し、説明者による正誤判定も一時停止する。既存の解答キューと解答権は保持され、フロー完了後に同じターンを再開する。

7.4.1 投票が同票のとき

⚖️
可決票と否決票が 同数 の場合は 否決 扱い（指摘不成立）。指摘者から 1 点減点。

7.4.2 投票の確定条件（MVP）

🗳️
MVP では投票タイマーを設けない。オンラインの投票権者全員 が投票した時点で確定。投票中に切断した参加者は一時的に投票権者から除外。再接続後は投票権者に戻る。オンライン投票権者が 0 人になった場合は 保留 とし、ホストは得点変動なしで指摘フローを無効化して同じターンを再開できる。

7.5.1 説明者によるスキップ

⏭️
説明者のみがスキップ可能（通常ターン進行中・指摘フロー中を除く）。スキップ時は加減点なし、解答キューをクリアして次ターンへ進む。ホストや解答者はスキップ不可。

7.6 説明時間（MVP）

⏱️
MVP では各ターンの説明パートに対するアプリによる 時間制限・時間切れによる強制ターン進行は行わない。説明の区切りは正解・スキップ・指摘フロー・参加者の運用に委ねる。
将来機能 説明時間・時間切れの導入は将来機能。

7.7 得点配分と正解時の加点

+1
正解した解答者
+1
当該ターンの説明者
−1
指摘成立・可決時の説明者
−1
指摘否決・同票時の指摘者
✅
両者加点（説明者にも加点）を採用。説明の成功も得点に反映されるため、全員が説明者のターンを真剣にやりやすく、説明と解答の両方で盛り上がる。

7.8 ルーム参加・ホスト

項目仕様
参加方法MVP では ルームコード入力 と URL 参加 の両方を提供する。参加者は どちらか一方 の方法で参加できればよい。
ゲーム開始ホスト（ルーム作成者）のみ。オンライン参加者 3 人以上 の場合に限り開始できる。
ホスト権限原則ルーム作成者がホスト。
ホスト切断切断された場合、現在接続中の参加者 のうち 入室が最も早い参加者 へ自動移譲。
手動移譲ホストは 任意の参加者 にホスト権限を手動で移譲できる。

7.8.1 入室時のプレイヤー設定

🚪
入室完了条件は プレイヤーネームの入力・検証の完了のみ。手描きアイコンの確定を待って入室をブロックしない（アイコン確定まで入室できないと参加者の把握が困難になるため）。

7.8.2 プレイヤーネームの入力規約

項目仕様
必須1 件必須。未入力で入室完了は不可。
前後の空白先頭・末尾の空白は削除（trim）してから検証・保存。
空白のみtrim 後が 0 文字の場合は無効。
最大文字数trim 後 32 文字以下（Unicode コードポイント単位）。
最小文字数trim 後 1 文字以上。
制御文字U+0000〜U+001F、改行等は 除去して保存。trim と制御文字除去後に 1〜32 Unicode コードポイントであれば有効。
同一ルーム内の重複許可する（識別は名前・アイコン・内部 ID で行う）。
入室後変更MVP では入室後変更不可。将来機能

7.8.3 手描きアイコンの扱い

✏️
編集タイミング
入室後（ロビー・待機画面中）のみ編集可。ゲーム開始後は固定。
⬜
空白でもOK
空白キャンバス（未描画）のまま確定してもよい。未設定はプレースホルダー表示。
🚀
開始への影響なし
全員のアイコン確定をゲーム開始の必須条件としない。プレースホルダーのまま開始可。
●
描画 UI
正方形キャンバス、白背景、黒ペンのみ。ペン太さは固定。消しゴムとクリアを提供する。
PNG
画像仕様
256×256 PNG、最大 100KB。未描画でも保存可。上限超過はサーバで拒否する。

7.8.4 再接続

🔄
入室時にサーバが participant_token を発行。クライアントはブラウザ内に保存し、再接続時にサーバへ提示。サーバはトークンを検証し同じ参加者として復帰させる（得点・ホスト権限・解答キュー・投票権を引き継ぐ）。

7.8.5 ゲーム中の切断

対象仕様
説明者同じターンを一時停止し、説明者の復帰を待つ。ホストは得点変動なしでターンをスキップできる。
解答権者解答キューから自動削除し、次の参加者へ解答権を移す。キューが空の場合は通常ターンに戻り、次の解答ボタン待ちとする。
ホスト現在接続中の参加者のうち、入室が最も早い参加者へホスト権限を自動移譲する。

7.9 スコア・順位

🏆
順位決定
全ラウンド終了時点の得点で順位付け。
🤝
同点
同点の場合は 同順位でよい（タイブレークなし）。
📉
得点の下限
下限なし。指摘失敗等により −1、−2 等の負数を許容。

将来機能 ユーザー独自リストの登録・選択。

8.2 参加者プロフィール

データ形式タイミング
プレイヤーネーム文字列（最大 32 コードポイント）入室時に必須入力
手描きアイコン256×256 PNG（最大 100KB）。白背景・黒ペンのみの正方形キャンバスから書き出し入室後に編集・確定（空白でも可）

セッション内で全員の画面に同期。MVP ではロビー中のアイコン更新をリアルタイム反映。ゲーム開始後は固定。

9 非機能・技術方針（概要）

項目方針
同期セッション型: 1 ルーム = 1 ゲーム、全員が同一状態を参照するリアルタイム同期。
クライアントWeb（PC・スマホ）。各人 1 端末を基本とした UI。
音声本アプリは通話を提供しない。Discord 等は利用者が別途利用する。
認証・永続化MVP ではアカウント認証は必須としない。participant_token による匿名参加・再接続。
手描きアイコンCanvas API 等による描画、タッチ／ポインタ対応（スマホ・PC）。画像の転送サイズに上限あり。
