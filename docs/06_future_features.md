# 06 — 将来機能

本書は [01_requirements.md](./01_requirements.md) の MVP から外す機能や、将来の拡張候補をまとめる。MVP の実装判断は `01_requirements.md` を優先し、本書は後続フェーズの検討材料として扱う。

---

## 1. 将来機能の候補

- **2 人プレイモード**（投票なし、指摘の扱いを別ルール化など）。
- **ライフ制**（得点とは別にライフを管理し、失格やタイブレークに利用する）。
- **説明時間・時間切れ**（各ターンの説明パートに制限時間を設ける）。
- **投票時間**（否認後の投票に制限時間を設ける）。
- **ユーザー独自お題リスト**の登録・共有。
- **説明者向け**: 長時間無操作時の **アラート**・**自動不正解**。
- **音声を補助する**文字起こし・カタカナ検出（参考表示のみ、判定は人間優先など）。
- **アイコン描画の省力化**（プリセットスタンプ、テンプレート枠内のみの着色、画像ファイルの直接アップロードなど）。
- **音声通話の本アプリ内完結**（WebRTC 等によるルーム内ボイスチャット）。

---

## 2. 2 人プレイモード

MVP では 3 人以上をゲーム開始条件とする。2 人プレイは、説明者と指摘者を除いた投票者が存在しないため、指摘の成立条件を別ルールとして設計する必要がある。

検討例:

- 投票を行わず、説明者の是認のみで指摘成立とする。
- 指摘をスコア処理ではなく、会話上の注意として扱う。
- 指摘ルールを無効化し、正解・スキップのみで進行する。

---

## 3. ライフ制

ルーム作成時に **ライフ制あり**のように有効化する想定。MVP では [01_requirements.md §6](./01_requirements.md#6-ルーム作成設定)・[§7.9](./01_requirements.md#79-スコア順位) のとおりライフは用いない。

### 3.1 二つのステータス

| ステータス   | 役割                                                                                                                                                                                                 |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ポイント** | **正解**等により加算する（減点は [01_requirements.md §7.4](./01_requirements.md#74-指摘と投票) 等の既存ルール）。                                                                                  |
| **ライフ**   | 初期値はルーム設定で与える（数値は実装時に定める）。**指摘成功**（**是認**、または投票で **可決**され、当該ターンの **説明者**がカタカナ違反とされた場合）に **説明者のライフを減少**させる。 |

### 3.2 失格

- **ライフが 0** になった参加者は **その時点で失格**する。
- **得点（ポイント）がいくらであっても**、失格後は **通常の順位付けの対象外**とする（「失格」として扱う）。

### 3.3 順位付け

- **失格していない**参加者のうち、まず **得点が高い**順に並べる。
- **得点が同じ**場合は、**残りライフが多い**方を **上位**とする（タイブレーク）。
- 失格者同士の順序や表示方法は実装時に定めてよい。

### 3.4 既存ルールとの関係

指摘可決時の **得点減**（[01_requirements.md §7.4](./01_requirements.md#74-指摘と投票)）と **ライフ減**を **併用**するか、ライフ制では **どちらか一方**にするかは、実装・バランス調整時に決定する。

---

## 4. 説明時間・時間切れ

ルーム作成時に **各ターンの説明パート**に対する時間制限（分／秒）を **設定可能**とする想定。MVP では [01_requirements.md §7.6](./01_requirements.md#76-説明時間mvp) のとおり採用しない。

- 時間制限の対象は **各ターンの説明パートのみ**とする（解答・投票の個別タイマーは別途検討）。
- **時間切れ**になった場合:
  - 当該 **ターン** の説明は **終了**し、**次のターン** に進む（**次の説明者** に交代し、**お題は新しいワードに更新**される）。
  - 同一ラウンド内の **最後のターン** の後は、**次のラウンド** の 1 ターン目へ進む（全ラウンド終了時はゲーム終了へ）。
- 細かい遷移（時間切れ直後に **解答キュー**をクリアするか、指摘・投票中は一時停止するか等）は実装時に明記する。

---

## 5. 投票時間

ルーム作成時またはゲーム開始前に、否認後の投票に対する制限時間（秒）を設定可能にする想定。MVP では [01_requirements.md §7.4.2](./01_requirements.md#742-投票の確定条件mvp) のとおり投票タイマーは用いない。

- 投票時間を有効にした場合、投票は **オンラインの投票権者全員が投票した時点**、または **投票時間が切れた時点**で確定する。
- 時間切れ時は、その時点で集まっている票で可決／否決を判定する。可決票と否決票が同数の場合は、[01_requirements.md §7.4.1](./01_requirements.md#741-投票が同票のとき) と同じく **否決扱い**とする。
- 投票時間導入時の切断者・未投票者の扱いは、MVP のオンライン投票権者ベースの考え方を前提に、実装時に詳細化する。

---

## 6. ユーザー独自お題リスト

MVP はサーバ側の固定リストを利用する。将来は、ユーザー独自リストの登録・選択・共有を追加できる。

検討例:

- ルーム作成時に固定リスト／独自リストを選択する。
- 独自リストをユーザー単位、または共有 URL 単位で管理する。
- お題数不足、重複、禁止語、公開範囲の扱いを定義する。

---

## 7. 説明者向け補助

MVP では、説明者の正誤操作が長時間ない場合も自動タイムアウトは行わない。将来は、アラート表示や自動不正解などを検討する。

---

## 8. 文字起こし・カタカナ検出

音声を補助する文字起こしやカタカナ検出は将来機能とする。採用する場合も、ゲームの判定は人間優先とし、検出結果は参考表示に留める方針を検討する。

---

## 9. アイコン描画の省力化

MVP では [01_requirements.md §7.8.3](./01_requirements.md#783-手描きアイコンの扱い) に従い、描画 UI を経由した手描きアイコンを扱う。空白キャンバスでの確定は許可する。

将来は、プリセットスタンプ、テンプレート枠内のみの着色、画像ファイルの直接アップロードなど、描画 UI を経由しない登録方法を別仕様で定義する。

---

## 10. アプリ内音声通話

MVP では [01_requirements.md §2.2](./01_requirements.md#22-アプリの担当範囲)・[§9](./01_requirements.md#9-非機能技術方針概要) のとおり **音声は外部**とする。

将来は、Discord 等の外部ツールに頼らず、本アプリだけで説明・雑談・投票時の会話を完結させる案を検討できる。例として、WebRTC によるルーム内ボイスチャット、ミュート、話者表示などがある。インフラ、プライバシー、帯域、録音の有無は別途検討する。

---

## 11. データ・API 拡張メモ

将来機能を採用する場合の拡張例:

| 機能                 | データ・API の拡張例                                                                 |
| -------------------- | ------------------------------------------------------------------------------------ |
| ライフ制             | 参加者に `lives`、ルーム設定にライフ制の有効／無効と初期値を追加する。               |
| 説明時間             | ルーム設定に説明タイマー秒数、ターン状態に期限時刻または残り時間を追加する。         |
| 投票時間             | ルーム設定に投票タイマー秒数、投票状態に期限時刻を追加する。                         |
| ユーザー独自お題     | お題マスタを固定リストからユーザー単位または共有単位のリストへ拡張する。             |
| プロフィール編集     | 入室後のプレイヤーネーム変更を許可する場合、表示名変更イベントと履歴扱いを定義する。 |
| アプリ内音声通話     | 音声セッション、接続状態、ミュート状態、権限、シグナリング API を追加する。           |
