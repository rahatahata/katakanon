# 05 — サイトマップ・画面一覧

本書は [01_requirements.md](./01_requirements.md) に対応する **画面・URL** の案を整理する。ルーティングは実装フレームワークに依存する。

---

## 1. 公開ルート（案）

| パス（例）      | 画面名                        | 説明                                                  |
| --------------- | ----------------------------- | ----------------------------------------------------- |
| `/`             | トップ / ルーム作成・参加入口 | 新規ルーム作成、ルームコード入力、参加 URL からの着地 |
| `/r/{roomCode}` | ルーム（ロビー〜ゲーム）      | 同一 URL 内でフェーズ遷移しても、パス固定でも可       |
| `/help`         | 遊び方（任意）                | MVP 省略可                                            |

---

## 2. 画面フェーズ（単一ルーム内）

要件 [§7.1](./01_requirements.md#71-全体フロー) に沿った **論理画面**。

| フェーズ          | 主な UI                                                                                                          |
| ----------------- | ---------------------------------------------------------------------------------------------------------------- |
| **ロビー / 待機** | 参加者一覧、プレイヤーネーム表示、手描きアイコン編集、ルーム共有、ホストの「ゲーム開始」（3人未満は不可）        |
| **ゲーム中**      | 現在ターンの説明者・解答者の表示、お題（説明者のみ）、解答ボタン、解答キュー表示、説明者の正誤・スキップ、スコア |
| **指摘 / 投票**   | 指摘ボタン（排他）、解答操作の一時停止、説明者の是認／否認、否認時は投票 UI、投票保留表示                        |
| **結果**          | 最終順位、同順位表示（要件 [§7.9](./01_requirements.md#79-スコア順位)）                                          |

---

## 3. モーダル・オーバーレイ（案）

- ルームコード共有（コピー）
- ホスト移譲・切断時の通知
- ゲーム開始不可（3人未満）の通知
- エラー・再接続（要件 [§7.8.4](./01_requirements.md#784-再接続)）

---

## 4. レスポンシブ

- **PC** — 一覧・キューを横並びにしやすいレイアウト。
- **スマホ** — タッチ、Canvas によるアイコン描画（要件 [§9](./01_requirements.md#9-非機能技術方針概要)）。
